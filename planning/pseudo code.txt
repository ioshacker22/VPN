Assignment #2: Secure VPN Application

In this assignment, you will design and implement a rudimentary Virtual Private Network (VPN) application.
The VPN will allow a client to securely connect to a server, authenticate, establish an encrypted tunnel, and exchange data safely over an untrusted network.

The application will be modular, extensible, and designed using object-oriented principles, templates, and composition, similar to a real-world VPN architecture.

VPN Application Overview

This application simulates a secure VPN tunnel between a client and server.

The VPN supports:

Authentication

Encryption

Secure data tunneling

Multiple cipher and protocol strategies

All global variables, functions, operator overloads, and types must be placed inside the seneca namespace.

tester Module 

Validates authentication flow

Simulates encrypted traffic

Verifies correct teardown

Do not modify this module.
Study the tester carefully.

crypto Module 

Implements cryptographic primitives:

Key generation

Encryption

Decryption

Supports multiple encryption strategies.

Do not modify this module.

socket Module 

Handles:

TCP/UDP socket abstraction

Data transmission

Connection lifecycle

Do not modify this module.

packet Module 

Defines the structure of VPN packets:

Headers

Payload

Integrity checks

Do not modify this module.

connection Module
Defines the interface that all VPN connections must implement.

Key virtual functions:

connect()

authenticate()

sendData()

receiveData()

disconnect()

ConnectionTpl Module

Implement a templated class ConnectionTpl<T> derived from Connection.

Template Parameters

T: encryption strategy (e.g., AES, ChaCha)

Private Members

m_socket

m_cipher

m_isAuthenticated

m_sessionKey

Public Members

Constructor initializing socket and encryption

connect() override

authenticate() override

sendData(const std::string&)

receiveData()

disconnect()

All data transmission must be encrypted using T.

VPNClient Module

Implement a concrete class VPNClient derived from ConnectionTpl<T>.

Private Members

m_username

m_password

Public Members

Constructor accepting credentials

authenticate() override:

Sends credentials securely

Validates server response

clone() override

VPNServer Module

Implement a concrete class VPNServer.

Private Members

Dynamic array of active connections

Authentication database

Public Members

Rule of 5

acceptConnection(Connection*)

removeConnection(const std::string&)

broadcastEncryptedData(const std::string&)

Tunnel Module

Implements the secure data tunnel.

Responsibilities

Encapsulate encrypted packets

Ensure integrity

Manage session lifetime

Public Members

openTunnel()

closeTunnel()

sendPacket(Packet&)

receivePacket()

Protocol Module

Implements VPN protocols (Strategy Pattern).

Example protocols:

OpenVPN-like

WireGuard-like

Each protocol defines:

Handshake logic

Rekeying strategy

Keepalive behavior

User Module

Represents an authenticated VPN user.

Private Members

Username

Access level

Session state

Public Members

Login

Logout

Permission checks

VPNManager Module

High-level orchestrator.

Responsibilities

Start server

Manage clients

Enforce security policies

Shutdown cleanly
